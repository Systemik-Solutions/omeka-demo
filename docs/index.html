Modules

Custom Ontology

Create specific classes and properties to describe resources when no standard ontologies are available, in particular on schema.org and in other places. It is useful to manage internal properties, or for researchers who are creating new data. The properties and classes are available via the standard API of Omeka S too, like any other ontology.

Repository Project Example

CSV Import

Import Entities from a simple CSV (comma separated value) file, and then map the CSV column data to Entity.

Repository Project Example

Custom Vocab

Create custom vocabularies and add them as data types to your resource templates.

Repository Project Example

Data Types

3 modules (Data Type Geometry, Data Type RDF, and Numeric Data Types) for adding new data types for the values of properties.

Project Example

Node Graph

Create interactive node graph visualizations with flexible configuration for item selection, grouping, styling, and relationship mapping.

Repository Project Example

Glycerine IIIF Viewer

Embed IIIF manifests using the lightweight Glycerine Viewer. This module supports both automatic rendering from item metadata and manual embedding via site blocks.

Repository Project Example

Mapping

This module expands the mapping functionality in Omeka S by adding new configuration options and visualization modes. It allows users to map not only attached or queried items, but also their linked items, group items with custom colors, and visualize journeys of geo-located resources.

Repository Project Example

Zotero Import and Zotero Citations

Two modules to import items from Zotero, and to use Zotero in rich-text editors to generate citations and bibliographies.
